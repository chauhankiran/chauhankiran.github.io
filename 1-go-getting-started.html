Go is programming language initially design and developed by Ken Thompson, Rob Pike and Robert Griesemer at Google, Inc. Go is now open sourced and community driven programming language.

Installation

In order to follow this course (and specialization), you need Go on your system. You might have Debian based system such as Ubuntu, Mint or elementary OS. Or you might have RHEL based system such as Red Hat, Fedora or CentOS. Or being a nerd, you have chosen Arch. No matter what GNU/Linux distribution you have, Go is available on almost all GNU/Linux systems. Even every popular package managers have package entry for Go. For example, Debian supported apt have golang-go package that contains Go language. You can easily install it by running following command. 

$ sudo apt install golang-go

RHEL supported dnf have golang package that contains Go language. You can easily install it by running following command.

$ sudo dnf install golang

Finally, Arch supported pacman have go package that contains Go language and many more things related to Go. You can easily install it by running following command.

$ sudo packman -S go

You can confirm the installation of Go on your system by running go version command in Terminal (name of command-line program).

$ go version
go version go1.15.6 linux/amd64

I've go1.15.6. If you get similar output (don't worry, if you get different version) then congratulation, you are ready for next step - writing Hello, World program.

Hello, World Program

Let's start the journey with Go programming language by touching out the hello.go program in Terminal. Yes, Go language has .go extension. 

$ touch hello.go
$ atom .

Here, I have created hello.go program and then open the Atom text-editor in current directory by running atom . command.

You know Go is a programming language but the compiled one. So, Go is not just a programming language but a Go is compiled programming language. As usual with any other traditional compiled programming language, it must have main() function to start the compilation process.

Note: function or method? For Go, its function.

Let's add this main() function in our hello.go program. 

func main() {

}

Yeah! Go has func keyword for function (JavaScript has function, Rust has fn and Lox has fun for functions!)

Note: Have you notice this curly braces? Go is highly influenced by C language (actually the initial developer and designer of Go - Ken Thompson was best friend of C's creator Dennis Ritchie. "Was"? Yes! Because Dennis is not with us - 1941-2011).

Go compiler will execute this main() function as compilation process. But, this main() function is empty and we need to print the famous Hello, World text on screen. Let's add Printf() function with this text in our hello.go program (P is capital in Printf()). 

func main() {
  Printf("Hello, World\n");
}

Alert: I don't think you need any explanation on the usage of \n. If you need then I am afraid that you does not satisfies the prerequisite of this article.

But, this Printf() is not a language construct. It's part of Go's fmt package. Actually, package is just a fancy word for standard library. So, we need to import/include this fmt package first in our hello.go program to use this Printf() function. This inclusion can be done by writing import statement in following way: 

import "fmt";

func main() {
  Printf("Hello, World\n");
}

That blank line is just for readability purpose. Also, fmt or formatted.

Now, we need to update our usage of Printf() function with fmt.Printf() because this function is part of fmt standard library or package. So, our hello.go program becomes 

import "fmt";

func main() {
  fmt.Printf("Hello, World\n");
}

Finally, add this package main line at the top of our program in following way. 

package main;

import "fmt";

func main() {
  fmt.Printf("Hello, World\n");
}

This package main statement automatically make a sense for you once you become familiar with Go's recommendation on code structure.

So, this is our Hello, World program in Go. It is time to run our hello.go program using go run hello.go command in Terminal. 

$ go run hello.go
Hello, World

Beautiful! We get the desire output! Nothing make us happy then a program without errors and warnings!

Here, We have used go compiler (because, Go is compiled programming language) and passed the name of file hello.go with run command. Meaning, Hey go! Can you please run hello.go program and give me the output?

In future articles, we will talk about more command, go compiler supports. But for now, just remember this command - go run name_of_file.go.

That's it for Hello, World program. I'm giving you next 5 minutes to write and run Hello, World program on your system to confirm the installation and run time environment.

Looks like everybody has successfully write and run this Hello, World program. Before we talk about next topic, I want to clean this Hello, World program little bit and we are going to use this cleanness throughout the specialization.

Semicolons are optional in Go if you are writing single statement per line. That's what we did in our hello.go program, right? So, that we can remove all three semicolons from our program. Let's do that - 

package main

import "fmt"

func main() {
  fmt.Printf("Hello, World\n")
}

And if we run this program, 

$ go run hello.go
Hello, World

See, we get the identical output. This is the cleanness we are going to use.
