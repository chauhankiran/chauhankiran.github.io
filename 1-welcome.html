1. Welcome

In this series, we're going to build the CSS framework from the scratch. We'll start from nothing and going to build a CSS framework that you can use in your next side project with more or fewer modifications. In other words, you'll have the CSS framework in your hands!

Before we go further, I would like to discuss few points.

  1. We're going to use CSS language to build this CSS framework. I know that we've preprocessors such as Sass, Less, or Stylus. I also know that these preprocessors are great for this specific purpose. But, I'll not use them. Just simple and plain CSS language. Why? Because, I want to minimize the barrier to follow this series. I mean if you're planning to read these articles, chances are high that you know CSS. But, I can't assume similar for these preprocessors. Even it might possible that you haven't heard about them at all. Still, towards the end, we'll port the framework using Stylus.
  2. The framework that we'll build in this series, will be small and will have a minimum set of features/components. You wouldn't find many features in this framework and that is intentional. Because, we're not here to build a framework that will compete with popular frameworks such as Bootstrap or Tailwind CSS. The framework we're going to build in this series is for fun and education. (That is more important I think).
  3. Finally, The framework will not include a single line of JavaScript. Because, we're building a style framework and it should only have styles, right? It may or may not have JavaScript! But, yeah! we'll write the minimum JavaScript code that might require for a component to work properly.

I'm planning to divide the series into four parts (But, let's see how things go).

  1. In the first part, we're going to build the framework itself - v0.0.1.
  2. In the second part, we'll build the ecosystem around the framework by creating different sets of themes and add-ons. We'll also report the problem(s), if we find it while creating these themes or add-ons.
  3. In the third part, we'll enhance our framework by fixing the problems mentioned in part two and also going to add few more components. Then we'll release v0.0.2.
  4. Finally, in the fourth part, we'll put more importance on open source by providing the documentation, development guide, contribution guidelines, and port the framework using Stylus.
