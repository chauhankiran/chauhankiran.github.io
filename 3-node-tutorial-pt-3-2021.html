Where we're? Aha, you asked a question and I told you that I'll answer your question later. Now, it is time to answer your question. Let me first copy the question here.

I have not require() console and still I was able to access .log() method. But, why not .platform() method?

We tried, but .platform() method didn't work without requiring the os module. Let me surprise you more with another method - .cwd() from process module. Create a new file with name process.js and write the following code. 

// A simple program that give current working directory/folder.

let cwd = process.cwd();
console.log(cwd);

If you run this program, you'll get the following output.

$ node process.js
/home/kiran/Code/Node

In this program, I invoked .cwd() method from process module and saved the return value into variable called cwd. Based on documentation, this method

returns the current working directory or folder of the Node.js process.

In simple words, the folder location of process.js from where this file is executed by Node. After that we consoled out the return value. All these works without requiring process module!

Now it is time to reveal the secret. The console and process modules work without require() statement because they are the part of globals. Yes, all the methods, objects or constants define as globals are accessible in any Node program within explicit inclusion. Actually, require() itself listed under globals module and hence we don't have to worry about where it comes and what to write - just use require() statement. So, in future, while reading someone else code and found process.env without inclusion of process module, don't surprise, you now know, right!

Let's create new file with name globals.js and write the following code. 

// A simple program that give current directory and file name using variables.

console.log(__dirname);
console.log(__filename);

Running this program print the following output.

$ node globals.js
/home/kiran/Code/Node
/home/kiran/Code/Node/globals.js

__dirname is a global constant that you can use directly in your program and it'll print the current working directory or folder. There is different between this __dirname and process.cwd() that I will explain you later in future articles while write practical solutions. But, if you're curios to know then you can read this or this answer to understand the difference. Finally, __filename is a global constant that print the name of file in execution.

I hope you know understand that there is nothing magic or exceptional here. We're not requiring those modules because they're already there as part of globals.

That's it for the globals and while we're here in standard library, we're going to explore few other modules to become familiar ourself with Node's code ecosystem. See you in the next article!
