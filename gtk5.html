The first point is, the comment is outdated in our hello-gtk.c program. I mean we were so busy updating our code that we almost forgot to update the comment and now comment contains information that is not relevant to the code. This is one of the classic programming problems, where you start with something idiomatic and then you become so busy or excited with further coding that you just forgot or don't care about the comment and/or documentation that make it deprecated or outdated. That's exactly what happened with our hello-gtk.c program. We were so busy with adding support for terminating the program that we forgot to update the comment (even worse, we haven't written a comment on the function we defined). Let's resolve this problem by updating the comment in our hello-gtk.c program.

First thing first, let's remove the existing comment because it doesn't comply with written code.

The second point is, I don't know about you, but I'm now tired of writing that big compilation command every time I do a modification or even a little change in my code. I heard about something called Makefile. I'm not Makefile Ninja but I just learned the basic syntax of Makefile and how to use it. It is quite simple.

Before you follow these steps, be sure that you set (maybe for now only) tab to its default size (possibly 8) and you are not converting the tab into spaces. That being said, create a file with name makefile (no extension) in a folder where you saved hello-gtk.c. Once created, write the following code inside the makefile. 

compile:
        gcc hello-gtk.c `pkg-config --cflags --libs gtk+-3.0` -o hello-gtk

This is called rule in makefile. Now, instead of writing this whole line while compiling the hello-gtk.c program, we just need to write the command make in terminal. This command executes the first rule written inside this makefile which is in our case compile. 

$ make
gcc hello-gtk.c `pkg-config --cflags --libs gtk+-3.0` -o hello-gtk

For example, I just ran the make command and it executes what is written in tabbed code as a command. Perfect! New, I just need to write make only and it will make an executable file for me.

Although it is not required, if you want, you can hide the command printing in the terminal by prefixing the command with @ in makefile rule. 

compile:
        @gcc hello-gtk.c `pkg-config --cflags --libs gtk+-3.0` -o hello-gtk

Now, if I run the make command, it'll run but not print the actual command assigned to it.

Now, if I run the make command in the terminal, it'll compile the program but not print the command it runs. Normally, when you compile the program it replaces the previously created executable with a newer one. So, most of the time, you wouldn't delete that executable. But, we can also write a rule for deleting the old executable in a makefile You know the command to delete or remove the (executable) file - rm, right?. 

compile:
        @gcc hello-gtk.c `pkg-config --cflags --libs gtk+-3.0` -o hello-gtk

clean:
        rm hello-gtk

Let's print that we're removing the executable using echo (it is good to print this operation because we're doing destruction). 

compile:
        @gcc hello-gtk.c `pkg-config --cflags --libs gtk+-3.0` -o hello-gtk

clean:
        echo "removing the executable..."
        rm hello-gtk
        echo "removed the executable."

To run this rule, you have to write make clean because by default it runs the first rule when you don't specify the rule name. Except first rule, you have to specify the name with make like make clean. 

$ make clean
echo "removing the executable..."
removing the executable...
rm hello-gtk
echo "removed the executable."
removed the executable.

The output is little clumsy. Lets not print the actual echo command because we're already printing the message. You know how to do that right? By prefixing @. 

compile:
        @gcc hello-gtk.c `pkg-config --cflags --libs gtk+-3.0` -o hello-gtk

clean:
        @echo "removing the executable..."
        rm hello-gtk
        @echo "removed the executable."

Now, if we run the command, it will print an informative message like ( also if you get any error while running the make clean, you need to run make command to create an executable because if you don't have an executable, how you clean it!) 

$ make clean
removing the executable...
rm hello-gtk
removed the executable.

This log says that it started removing the executable, then it tells us which executable removed with a command that is used, and finally, it prints the message the operation is completed.

This is not the perfect makefile but I think it helps us to reduce some of our manual work. So, it is okay to use it for now in the current format. As I learn more about Makefile, I will update based on it.
