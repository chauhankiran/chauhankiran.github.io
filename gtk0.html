GTK+ or GTK is GUI toolkit (written in C) for developing GNU/Linux (or other platforms, if you care) desktop applications. There are two popular versions of GTK - 2 & 3. 2 is stable but old. 3 is new but occasionally breaks! (even version 4 is coming). The focus of this book is - 3 and in C. Due to this, this book assume that you know C.

Here is the simple hello, world program (hello.c) in C. 

#include<stdio.h>

int main(int argc, char *argv[]) {
  printf("hello, world\n");

  return 0;
}

To compile this program, we're going to use gcc. Following is a command to compile this program. 

$ gcc hello.c -o hello

gcc is a command that invoke compiler on hello.c and after successful compilation, it will generate the object file (-o) hello. This object file can be execute using following command: 

$ ./hello
hello, world

But, we're talking about GUI applications in GTK. So, we first need to install the development libraries for GTK. Almost, all popular package managers have entry for GTK development libraries. You just need to run the appropriate command(s) to install these development libraries. Go ahead and do first! Then let's meet in next section.

In order to build GUI application using GTK, we need to include one header file gtk/gtk.h in our program. For example, if I'm creating hello, world application within hello.c, then following will be the first line I have to write (create a hello.c program or delete everything from hello.c, if you already have). 

#include<gtk/gtk.h>

After this, as obvious you need a main() function to initiate the execution. 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {

}

Optionally, we can return something at the end of main() function. 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {

  return 0;
}

Within this main() function, we first need to initiate the GTK using gkt_init() function. This function do all the needed configurations and initial setup before our GUI render on screen. This function takes passed parameters. 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {
  gtk_init(&argc, &argv);

  return 0;
}

Then we can write our application code that might have window, button, menu, icon, and so on. 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {
  gtk_init(&argc, &argv);

  /* application code should be written here */

  return 0;
}

Once we complete with out application code, we need to invoke, at the end but before the return statement - gtk_main() function. This method take the control of our application and continue run until we terminate the application or process. This is called main loop and continue run while having the control of application. 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {
  gtk_init(&argc, &argv);

  /* application code should be written here */

  gtk_main();

  return 0;
}

What we just wrote here is the boilerplate code for the GTK GUI application for next few chapters. In future chapters, whenever I ask you to write the boilerplate code, this is what you have to write. Let's clear the program and re-write these few lines again but now in little different order.

Let's first include the needed header file. 

#include<gtk/gtk.h>

Then write a main() function with return statement at the end. 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {

  return 0;
}

Next, let's initiate the GTK using gtk_init() function. 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {
  gtk_init(&argc, &argv);


  return 0;
}

Before we forgot, let's add the main loop at the end but before the return statement. 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {
  gtk_init(&argc, &argv);

  gtk_main();

  return 0;
}

Finally, we can write our application code between init function and main loop. 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {
  gtk_init(&argc, &argv);

  /* application code should be written here. */

  gtk_main();

  return 0;
}

Until we take different approach to build GTK applications in future chapters, you need to write this boilerplate code again and again until you frustrate with typing this too much! Because, that's how you understand, learn, and remember!

Note the name of both functions gtk_init() and gtk_main(). Every function in GTK starts with gtk_. You may try to write this boilerplate code couple of times until you remember. Once you done with it, let's meet in next chapter to write hello, world program using GTK.
