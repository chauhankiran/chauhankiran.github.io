Let's Build miv - Mini Image Viewer

Oct 2020

In this mini-book, we're going to build miv - mini image viewer using GTK3 and Vala. In order to follow this tutorial, you need

  • Vala compiler
  • GTK3 development libraries
  • Meson
  • Terminal
  • Text editor

I haven't mentioned the steps to install these software, but you can communicate with me in case of trouble while doing installation. That being said, let's get started.

Let's create a folder with name miv and inside this we're going to create a file with name Application.vala.

$ mkdir miv
$ cd miv
$ touch Application.vala

Let's open this Application.vala in your favorite text-editor and write the following code to test the Vala development environment.

int main(string[] args) {
  print("hello, world\n");

  return 0;
}

Vala is one of the language that requires main () method and semicolon at the end. To compile this program, following command is used.

$ valac Application.vala
$ ./Application

valac is a compiler that compiles the Vala program and after successful compilation, it'll generate executable with same name e.g. Application. We can easily run this executable by prefixing it with ./.

Let's import GTK library using using statement at the top of our Application.vala program.

using Gtk;

int main(string[] args) {
  print("hello, world\n");

  return 0;
}

Now, if we compile the program again, we'll get the following error.

$ valac Application.vala
Application.vala:1.7-1.9: error: The namespace name `Gtk' could not be found
using Gtk;
      ^^^
Compilation failed: 1 error(s), 0 warning(s)

Because, GTK is not standard library comes with Vala instead consider it as third-party library that you can use it in your program to develop GUI application! In such a case, you need to use --pkg option with valac as follow.

$ valac --pkg gtk+-3.0 Application.vala

And now, you should not see any error! After GTK import, you can use methods from GTK library such as init (). This method initialize all sort of essentials things for you to render GUI. It access the passed argument as ref.

using Gtk;

int main(string[] args) {
  init(ref args);

  print("hello, world\n");

  return 0;
}

At the end (but before return statement), we should call main () method and it'll continue execute our application.

using Gtk;

int main(string[] args) {
  init(ref args);

  print("hello, world\n");

  Gtk.main();
  return 0;
}

Play attention on how I call the main () method. instead of directly writing main () method I prefixed it using Gtk because it'll cause the problem if you get what mean!

You can even prefix the init () method as Gtk.init ().

using Gtk;

int main(string[] args) {
  Gtk.init(ref args);

  print("hello, world\n");

  Gtk.main();
  return 0;
}

If this is the case, we don't have to write using Gtk statement at the top and still compilation happen correctly.

int main(string[] args) {
  Gtk.init(ref args);

  print("hello, world\n");

  Gtk.main();
  return 0;
}

Between these init () and main () methods, you can write your application code. Let's create a simple window and display it.

Gtk.Window () class is used to create an instance of window widget. We also need to use new keyword to create an instance.

int main(string[] args) {
  Gtk.init(ref args);

  var window = new Gtk.Window();

  print("hello, world\n");

  Gtk.main();
  return 0;
}

Being static typed programming language, I have used var keyword to define a variable. Next, let's set the title using title attribute.

int main(string[] args) {
  Gtk.init(ref args);

  var window = new Gtk.Window();
  window.title = "This is window";

  print("hello, world\n");

  Gtk.main();
  return 0;
}

Finally, we can display it using show () method.

int main(string[] args) {
  Gtk.init(ref args);

  var window = new Gtk.Window();
  window.title = "This is window";
  window.show();

  print("hello, world\n");

  Gtk.main();
  return 0;
}

Let's remove this printf () method call as we do not needed it for now.

int main(string[] args) {
  Gtk.init(ref args);

  var window = new Gtk.Window();
  window.title = "This is window";
  window.show();

  Gtk.main();
  return 0;
}

This marks our simple window program in Vala using GTK. Let's run it and confirm our code.

$ valac --pkg gtk+-3.0 Application.vala
$ ./Application

You should see a small 200x200 window with title 'This is window' at top-left corner. If this is the case, then congratulation, you just created your very first Vala GUI program (the program is continue running in terminal which you can terminate using ctrl+c).

We can write our whole application within main () method and between Gtk.init () and Gtk.main () methods. But, that's not recommended way and you'll not see such type of code over the Internet. Why? Because, Vala is object-oriented programming language and due to this, developers utilize this feature to better manage their code.

Let's remove all the code we just wrote within Application.vala file and we're going to write the same code but this time with object-oriented programming.

Let's define a class with name Application (now you know what I named the file with this name!).

class Application {

}

We're going to extend this Application class with Gtk.Application because we're going to write GUI application using GTK! This inheritance or extend can be done using : operator.

class Application : Gtk.Application {

}

We extend our Application class with Gtk.Application class because this Gtk.Application class is the important one and it do many needed things for us such as GTK initialization, application uniqueness, desktop integration and many many more.

Let's write the default constructor which has same name as the class.

class Application : Gtk.Application {
  Application() {

  }
}

This constructor accepts two arguments via Object() as follow.

class Application : Gtk.Application {
  Application() {
    Object(

    );
  }
}

In this Object() we need to pass, application_id and flags. The value of application_id is string with unique name in most of the case reverse domain name and flags is GTK flag which is GLib.ApplicationFlags.FLAGS_NONE.

class Application : Gtk.Application {
  Application() {
    Object(
      application_id: "com.github.chauhankiran.test",
      flags: GLib.ApplicationFlags.FLAGS_NONE
    );
  }
}

GLib is already there for you to use in Vala and due to this, we can remove it.

class Application : Gtk.Application {
  Application() {
    Object(
      application_id: "com.github.chauhankiran.test",
      flags: ApplicationFlags.FLAGS_NONE
    );
  }
}
