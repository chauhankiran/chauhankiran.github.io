We're now going to write a simple hello, world program using GTK that display a blank window with title hello, world. This will be our first iteration of hello, world program in GTK. We'll going to write three sets of hello, world program in GTK with different concepts (or GUI components) each time!

Create a new program hello-gtk.c and write the boilerplate code. I'm going to repeat the boilerplate code steps with you as that will help with practicing.

First, we need to include the GTK library because this library contains all the functionalities related to GUI parts. 

#include<gtk/gtk.h>

But, that doesn't mean it gives us everything. Don't forget that we still need main() function. So, let's write a main() function with optional return statement. 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {

  return 0;
}

Next, we need to invoke gtk_init() function. This function will help us with all the initial configurations a GUI application need in order to render on screen. Don't forget to pass the command-line arguments to this function as follows: 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {
  gtk_init(&argc, &argv);

  return 0;
}

If we run this program, it might run from top to bottom and then exit or terminate. We don't want this! We want out GUI application to continue run so that it can response with our interaction. For example, if we click on button within our application, it should response instead of terminating the program after execution. To get this 'continue run' feature, we need to invoke gtk_main() function at the end, after everything render on screen (but before return statement as we discussed). 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {
  gtk_init(&argc, &argv);

  gtk_main();

  return 0;
}

Instead of adding that comment, let's add few blank lines so that we know where we have to type the application code: 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {
  gtk_init(&argc, &argv);




  gtk_main();

  return 0;
}

In GTK, everything is widget. The window you're seeing is widget, the button you're going to click is widget, the menu you've opened is widget and invisible container that you're not seeing but helping to manage other widgets to display side-by-side is widget. All these - window, button, menu, label are types or categories of widget. In normal C, you define a variable with type integer that might hold the discount as follows: 

int discount;

Similarly, we can to define a variable with type widget (GtkWidget), that might hold any of these mentioned (or not mentioned) widget. For example, 

GtkWidget *window;

Here, I defined a variable window. Now, this window variable is capable of holding any of the GTK widget. This GtkWidget is available for use from gtk.h header file.

You know where to write the above variable declaration code within hello-gtk.c file. 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {
  gtk_init(&argc, &argv);


  GtkWidget *window;


  gtk_main();

  return 0;
}

Just defining a variable with name window does not make it window. We actually need to create one using gtk_window_new() function. This function take one argument and until we understand this in great details with other arguments, just use GTK_WINDOW_TOPLEVEL as follow. 

window = gtk_window_new(GTK_WINDOW_TOPLEVEL);

This function create a window and return its pointer to window variable. Now, this window variable hold actual window!

Note: As I mentioned previously, every function with GTK normally starts with gtk_. Here is a next point for you to remember. If you're working with window, every function name should start with gtk_window_. After this prefix, function will have a verb that define its purpose. For example, we're creating new window so new. This will help us to remember function name like, we're using GTK(gtk_) window(window_) to create a newer(new) one.

Let's add this line in our hello-gtk.c program as well. 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {
  gtk_init(&argc, &argv);


  GtkWidget *window;
  window = gtk_window_new(GTK_WINDOW_TOPLEVEL);


  gtk_main();
  return 0;
}

So far we created a window variable with type of widget and then assign an actual window to this variable. Things are not end here. We need to write one more line to display this window on screen. To do this, we need to use gtk_widget_show() function. This function take one argument that is - the widget you're interest to show or display which is in our case window. Putting this explanation in code, 

gtk_widget_show(window);

Note: Again, the function is prefixed with gtk_. We're working with widget so gtk_widget_ and we're trying to show the widget so show hence gtk_widget_show().

This function will render window on screen. Let's add this line as well! 

#include<gtk/gtk.h>

int main(int argc, char *argv) {
  gtk_init(&argc, &argv);


  GtkWidget *window;
  window = gtk_window_new(GTK_WINDOW_TOPLEVEL);
  gtk_widget_show(window);


  gtk_main();
  return 0;
}

Let's remove the extra space and add some descriptive comments on what we wrote. 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {
  gtk_init(&argc, &argv);

  /**
   * 1. Create a variable that will hold a window.
   * 2. Create an actual window.
   * 3. Display the created window.
  **/
  GtkWidget *window;
  window = gtk_window_new(GTK_WINDOW_TOPLEVEL);
  gtk_widget_show(window);

  gtk_main();
  return 0;
}

Don't get me wrong but these comments are too trivial for seasoned GTK programmers. But, we're not seasoned GTK programmers and because of that we can write the comment like this for our understanding. In future, when we'll have enough experience, we might not write comments like this one!

We just wrote the first version of hello, world program in GTK. It is time to run this program and see what we get with these few lines of code!

You know how to run a simple program, right? 

$ gcc hello.c -o hello

But, our program is little different than usual C program because it includes the library other than standard one - gtk/gtk.h. Because of that we need to tell gcc to include this library in compilation. Otherwise gcc will complain about the functions such as gtk_init() and gtk_main() if not found via library. We can send such a message to gcc via pkg-config. pkg-config is a special software that handle the location and inclusion of any other library in compilation using following command: 

$ gcc `pkg-config --cflags --libs gtk+-3.0` hello-gtk.c -o hello-gtk

Here, we're compiling hello-gtk.c program using gcc and asking pkg-config to include gtk+-3.0 (the actual installed name of GTK development library on our system). If everything works, then after successful compilation hello-gtk object(-o) file will be created. You know how to run this executable, right? 

$ ./hello

You'll see 200x200 blank window with title hello-gtk. Wow! You just created a very first hello, world program in GTK that display a window - the holder of all widgets (at some context)! You can maximize or minimize this window. You can even re-position the window by moving from one place to another. You can even press close or x button to close the window. But, this will not actually close or terminal the process associated with this window. You'll see in terminal that the executable is still running. You need to press ctrl+c to explicitly terminate the process.

Before we go further and start working on next iteration. I've following two points:

  1. This hello, world program is not what we decided. In other words, we want title to be hello, world. Currently, it is displaying hello-gtk.
  2. When we close the window, it should close the process as well. What you think? Consider the scenario, where you opened a browser that took 678MB of RAM to run it with just two tabs and you're facing slow performance. You press the x or close button to close the browser but it is still running in background, Nightmare!
