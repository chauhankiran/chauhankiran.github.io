Node Tutorial Pt. 2 (2021)

JavaScript's popularity is almost completely independent of its qualities as a programming language. Douglas Crockford, JavaScript: The Good Parts

Let's start this article from where we left-off in previous one - hello, world program. Let me copy that previous hello-world.js program here so that we can discuss further. 

console.log('hello, world');

This program is identical to browser's hello, world program. You should know what is console and what is .log(). If not then let me briefly explain you that, console is an object provided by the JavaScript and .log() is the method from console object. Hence, we have written console.log(). Meaning that we're using .log() method from console object.

I also mentioned in previous article that the objects and methods which are related to browser wouldn't work in Node. For example, document.getElementById() or window.opener wouldn't work. But, the objects and methods which are mentioned in Node's official docs work. For example, os.platform() will work because it is mentioned in documentation for os module.

Why not write one program and confirm this method? Let's create a new program with name os.js and write following. 

// A program to demonstrate .platform() method from os module.

It is always a good idea to explain what you're doing via comment. Now, let's use .platform() method from os object or module as below. 

// A program to demonstrate .platform() method from os module.

os.platform();

From the docs, you can confirm that the .platform() method doesn't require any parameter(s). Hence, I have just written an empty (). I think, our program is ready to run. 

$ node os.js
/home/kiran/Code/Node/os.js:3
os.platform();
^

ReferenceError: os is not defined
    at Object.<anonymous> (/home/kiran/Code/Node/os.js:3:1)
    at Module._compile (node:internal/modules/cjs/loader:1108:14)
    at Object.Module._extensions..js (node:internal/modules/cjs/loader:1137:10)
    at Module.load (node:internal/modules/cjs/loader:973:32)
    at Function.Module._load (node:internal/modules/cjs/loader:813:14)
    at Function.executeUserEntryPoint [as runMain] (node:internal/modules/run_main:76:12)
    at node:internal/main/run_main_module:17:47

Woh! Looks like from the error, Node doesn't recognize os and complains that os is not defined.

We're getting this error, because although it is define within standard library as module, but by default it is not available to use. We need to include this os module in this program using require() method. How to include? Just write the following line at the top of our program. 

let os = require('os');

What we did here is, include the os module using require() method and save that reference of this inclusion within this os variable. Now, we can access all the methods within os module via this os variable. You can give any name to this variable os. But, it is recommended that you give the same name as the module you included. This will help you with reading program.

After adding this line in our os.js, our program is as follows. 

// A program to demonstrate .platform() method from os module.

var os = require("os");

os.platform();

Now, let's run this program again.

$ node os

What happen? Our program runs but we didn't get any output. Ah! Because, as per docs,

The os.platform() method returns a string identifying the operating system platform as set during compile time of Node.js.

Meaning that it returns string result not print it. Printing is a the part, that we have to do and we know one method to print something in terminal, right? Yeah, console.log() method. So, wrapping this, os.platform() within .log() method as follows. 

// A program to demonstrate .platform() method from os module.

var os = require("os");

console.log(os.platform());

Now, we are finally done with os.js program, promise! Let's run our program and it will surely give the result as follows. 

$ node os
linux

Ye! I got the linux as output because I'm using GNU/Linux operating system on my computer. You might get something else based on your operating system.

If you have sharp eyes and really paid attention to details then you might have question like, I have not require() console and still I was able to access .log() method. But, why not .platform() method? Both are the part of standard library of Node, aren't they?

Yes they are - console and os are both the part of standard library and you raised the valid point. Let's discuss this question in next article, because I have something else for you in this article.

As per the title, I'm writing the Node tutorial for 2021. What is the point, if I don't mentioned the latest improvements? That latest improvement that I want to talk in this article is, the usage of import...from statement.

You can use import...from statement instead of require() method. You might have question like what is this import...from statement, why should I use it and if it is required, then how can I use it? It is not possible for me to answer all these questions at this moment because I haven't even briefly explain you what is the import...from statement. But, the reason, I have to mention this now because that's what we're going to use in future once you become more familiar with Node and basic concepts. That's what the future code should look like!

Let's update out above os.js code with import...from statement instead of require() method. Let's create a new file with name os-latest.js and then copy the same code we wrote in os.js file as follows. 

// A program to demonstrate .platform() method from os module.

var os = require('os');

console.log(os.platform());

Perfect! Now write import .... from ....; after this require() method as follow. 

// A program to demonstrate .platform() method from os module.

var os = require('os');
import ... from ...;

console.log(os.platform());

Now, question for you, from where we're importing or what we're importing? os module, correct! So, let's write this after from as follows. 

// A program to demonstrate .platform() method from os module.

var os = require('os');
import ... from 'os';

console.log(os.platform());

Now, what we're importing from this os module? Everything. So, we can write * after import. But, I would suggest you to type os variable name in this way, we can import everything from os module under os variable. Just like we did in require() statement. So, let's update the ... after import with os as follows. 

// A program to demonstrate .platform() method from os module.

var os = require("os");
import os from 'os';

console.log(os.platform());

Finally, remove the require() statement line because we no longer needed it. 

// A program to demonstrate .platform() method from os module.

import os from 'os';

console.log(os.platform());

That's it! Our program is converted with latest import...from syntax.

Let's run the program to confirm the output! What we did here is the code improvement and the rule of thumb is, if you do code improvement, it should not effect the end-user output. 

$ node os-latest.js

import os from 'os';

Ow! We get the error with this import statement. But, the good thing about this error is, it also provide the possible solution and the solution is, set "type": "module" in the package.json or use the .mjs extension. We don't have package.json file for now so it is not possible to go with first solution. But, we surely update the extension of this file from normal .js to .mjs. Here, m for module. We created a JavaScript module and module can be imported.

Let's rename the os-latest.js to os-latest.mjs and it should have following code. 

// A program to demonstrate .platform() method from os module.

import os from 'os';

console.log(os.platform());

Now, let's run the program again! 

$ node os-latest.mjs
linux

The output is identical. But, our code has used the latest syntax to import the module.
