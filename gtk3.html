Let me copy the program we wrote in previous chapter. 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {
  gtk_init(&argc, &argv);

  /**
   * 1. Create a variable that will hold a window.
   * 2. Create an actual window.
   * 3. Display the created window.
  **/
  GtkWidget *window;
  window = gtk_window_new(GTK_WINDOW_TOPLEVEL);
  gtk_widget_show(window);

  gtk_main();
  return 0;
}

The first point that we need to solve is, to display the title hello, world. This can be done by using gtk_window_set_title(). This function takes two arguments - first the targeting window and second the title string which we want to put. Writing the explanation in code, it looks like: 

gtk_window_set_title(window, "hello, world");

But this will not work. Why? well, let's try and see the error! Before that what do you think, where we should the above line in our program? After the creation of the window and before displaying it. Because you can't set title before creating an actual window and it doesn't make any sense to set a title after the window is displayed! So, let's add this line at the location where we just confirmed! 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {
  gtk_init(&argc, &argv);

  /**
   * 1. Create a variable that will hold a window.
   * 2. Create an actual window.
   * 3. Display the created window.
  **/
  GtkWidget *window;
  window = gtk_window_new(GTK_WINDOW_TOPLEVEL);
  gtk_window_set_title(window, "hello, world");
  gtk_widget_show(window);

  gtk_main();
  return 0;
}

Note: Did you notice the consistent naming convention? First gtk_, then window_, and then verb - the actual action function is doing - set_title(). The reason why I'm saying this, again and again, is because in this way you can remember the function names.

You might or might not understand this error and a possible solution that this error has suggested. Pay attention to the lines/words which I highlighted in the above error log. First, the warning is saying that its incompatible type i.e. we passed a window with the type of GtkWidget which is not compatible or which is not needed by gtk_window_set_title() function. Next, it is saying that it expected GtkWindow but we passed GtkWidget.

Now, what is the solution to this error? The first possible solution to this error is, why not define a variable starting from the first as GtkWindow instead of GtkWidget? That's a good idea but then we'll face a problem with gtk_widget_show() as this function accepts GtkWidget type variable and not GtkWidow type variable. The second and valid possible solution is to why not convert the GtkWidget variable into the GtkWindow variable and then pass this converted GtkWindow variable to gtk_window_set_title() function?

To make this conversion or to cast we're going to use GTK's cast macros such as GTK_WINDOW - all in the capital letters. What this macro does, first it will check if the casting is possible or not, and second if it is possible, then it will do casting e.g. cast the variable from GtkWidget to GtkWindow. Don't be scared with this (little long) explanation. All in all, you just have to write - 

GTK_WINDOW(window);

and this macro cast the window variable from GtkWidget to GtkWidow variable. That's it! So, instead of passing the window directly, let's pass this with-cast window as follows: 

#include<gtk/gtk.h>

int main(int argc, char *argv[]) {
  gtk_init(&argc, &argv);

  /**
   * 1. Create a variable that will hold a window.
   * 2. Create an actual window.
   * 3. Display the created window.
  **/
  GtkWidget *window;
  window = gtk_window_new(GTK_WINDOW_TOPLEVEL);
  gtk_window_set_title(GTK_WINDOW(window), "hello, world");
  gtk_widget_show(window);

  gtk_main();
  return 0;
}

Now, if we run this program, it'll not produce the error and compiles successfully. Let's try it! 

$ gcc `pkg-config --cflags --libs gtk+-3.0` hello-gtk.c -o hello-gtk
$ ./hello-gtk

And this gives us a 200x200 default window with title hello, world, as we excepted. Good!

Now, before we hit the second problem, I have a couple of points that I want to discuss. The first point is about casting. You'll see the casting here and there, all over programs. The casting is simple as one macro call such as GTK_WINDOW(window) or GTK_WIDGET(window). The first one cast the window into the GtkWindow type and second one cast window into the GtkWidget type. The naming convention is also simple. GTK_, and the type into which we want to cast (also in the capital) e.g. WIDGET, WINDOW, and so on.

The second point is about declaring the variable with its initial type. There are a couple of reasons why we define the variable with type GtkWidget instead of any other type. I pointed out one possible reason while solving the error with get_window_set_title(). The second reason and most important one is the Widget Hierarchy. GTK is implemented in an object-oriented way and due to this, different types of widgets are created from one another by inheriting the features from existing once! This inheritance starts from the top and at the top, we have GtkWidget (except GtkObject). Because of this, if you define a widget with type GtkWidget, you'll get all the capabilities and functionalities that GtkWidget has, and a few more for your specific widget such as GtkWindow if you're planning to use GtkWidget variable as a window. And as always, you can back and forth between these two types by casting or not casting the variable.

Note: While searching on the Internet, you might find GTK+ in many places. The plus was added by Peter Mattis (co-creator of GTK/GTK+) while he rewrote GTK due to some architectural limitations. The new architectural design had a fuller type system with inheritance (sort of object-orientation).

The third and last point is about the meaning of object-oriented function calling in GTK. I mean, in GTK you do not have a dot operator or something to call upon via dot notation. Instead, if you want to call a function on one particular entity, you need to pass that entity as the (first) argument. That's what we did with gtk_window_set_title(). We want to set the title of the window, but which window? To clarify this, we pass the specific window as the first argument to this function. This is how calling function works in (object-orientated) GTK.
