Flask: An Introduction

We're going to learn Flask, a framework for building web applications using Python. Bookmark the official website as we're going to visit a lot.

Getting up and running with Flask is simple as running few commands in the terminal. The instructions written in this article are Windows-specific. If you're using other systems like macOS or Linux, you should have to read the Installation page for the commands you need to run.

Let's create a folder with the name hello-world, and then go inside of it. 

$ mkdir hello-world
$ cd hello-world

Create a virtual environment for this project to isolate the project's dependencies and versions. . 

$ py -3 -m venv venv

Once created, enable or activate the virtual environment using the following command. 

$ .\venv\Scripts\activate

After running this command, you'll see (venv) somewhere in your command prompt. In this way, you can confirm that the virtual environment is activated.

Note: If you're using PowerShell and if running the above command throws the SecurityError. Try to run Set-ExecutionPolicy RemoteSigned -Scope CurrentUser first and then run the above command. It should work.

Now, we can install the flask in our folder/application using the following command. 

$ pip install Flask

Let's wait for a few seconds to install the Flask and its dependencies. After the installation process is done, go ahead and open the hello-world folder in text editor. 

$ code .

Create a file with the name app.py.

$ touch app.py

Open the file in text-editor and import Flask from flask. 

from flask import Flask

Create an app instance using Flask(). 

from flask import Flask

app = Flask(__name__)

We're now ready to create the route in our web application.

Let's create / route and it should return the text hello, world when visited in the browser. 

from flask import Flask

app = Flask(__name__)

@app.route('/')
def index():
  return "hello, world"

When route / is visited in the browser, it'll run the associated index() method and return the text hello, world.

To run the application, we need to set one variable FLASK_APP in terminal using following command. 

$ set FLASK_APP=app.py

Now, running following command will auto run the app.py. 

$ flask run
 * Environment: production
   WARNING: This is a development server. Do not use it in a production deployment.
   Use a production WSGI server instead.
 * Debug mode: off
 * Running on http://127.0.0.1:5000/ (Press CTRL+C to quit)

As mentioned in the terminal output, the application is running at localhost and on port 5000. ctrl+click the URL from the terminal and it opens the mentioned URL in the browser. In the browser, you should see the hello, world text. If that is the case, you're Flask application and up and running! If not, then send me the error you're getting, I'll help you to resolve it.

We can return the HTML. Let's update the return statement with <h1> as follow. 

from flask import Flask

app = Flask(__name__)

@app.route('/')
def index():
  return "<h1>hello, world</h1>"

Now, reload the page in the browser.

Ah! The hello, world text is displaying as it is. It should display in header form. The reason is, we have to restart the server as we did the modification in the app.py. By pressing ctrl+c, we can quit the running server and then run the flask run command to run the server again.

To auto restart the server, just change the Environment to development. To do this, press ctrl+c first and then write the following command. 

$ set FLASK_ENV=development

This update the Flask environment to development. Now run the application again using flask run command. 

$ flask run
 * Environment: development
 * Debug mode: on
 * Restarting with stat
 * Debugger is active!
 * Debugger PIN: 125-688-527
 * Running on http://127.0.0.1:5000/ (Press CTRL+C to quit)

Notice the Environment in above terminal output? It is now changed to development.

Note: If you're using PS, you need to run $env:FLASK_ENV="development" command instead of mentioned above.

With this change in the application, we don't have to worried about restarting the server again.

We can return the JSON response using .jsonify() method from Flask. Let's first import this method with Flask in import statement. 

from flask import Flask, jsonify

app = Flask(__name__)

@app.route('/')
def index():
  return "<h1>hello, world</h1>"

Now, using this method, we can return the JSON object.

from flask import Flask, jsonify

app = Flask(__name__)

@app.route('/')
def index():
  return jsonify({ "message": "hello, world "})

As mentioned previously, we don't have to restart the server to get the latest update in output. Visiting the mentioned localhost URL should return now the above JSON response.

Flask nicely supports the dynamic route by mentioned it as a placeholder within angle brackets (<>). 

from flask import Flask, jsonify

app = Flask(__name__)

@app.route('/<id>')
def index():
  return jsonify({ "message": "hello, world "})

The mentioned placeholder of the dynamic route needs to be pass as a parameter of associated function and in our case, it's index() method with the same name e.g. id. 

from flask import Flask, jsonify

app = Flask(__name__)

@app.route('/<id>')
def index(id):
  return jsonify({ "message": "hello, world "})

This id parameter has the value of the dynamic route. So, we can use it in function if we want. 

from flask import Flask, jsonify

app = Flask(__name__)

@app.route('/<id>')
def index(id):
  return f"The id is {id}"

Here, I'm just printing whatever value we have for the id.

Visiting / throws a 404 Not Found error because we haven't the associated route for the same. The error message is the default from Flask and we can update it using, @app.errorhandler(404). 

from flask import Flask, jsonify

app = Flask(__name__)

@app.route('/<id>')
def index(id):
  return f"The id is {id}"

@app.errorhandler(404)
def file_not_found(error):
  return "4-0-4"

The associated function accept one arguments - error and it contains further information of the error such as code, name, and description.

Note: You can return this info to the user using error.code, error.name, and error.description. In case of error.code, you need to convert into a string using str() function as it returns the int.

redirect() method is used to redirect the URL for the given route. Let's add /about route and redirect it to /<id> route when visited in the browser. 

from flask import Flask, jsonify

app = Flask(__name__)

@app.route('/<id>')
def index(id):
  return f"The id is {id}"

@app.route('/about')
def about():
  return redirect('/github')

@app.errorhandler(404)
def file_not_found(error):
  return str(error.description)

Before we go further and test in the browser, let's not forget to import the redirect method from flask. 

from flask import Flask, jsonify, redirect

app = Flask(__name__)

@app.route('/<id>')
def index(id):
  return f"The id is {id}"

@app.route('/about')
def about():
  return redirect('/github')

@app.errorhandler(404)
def file_not_found(error):
  return str(error.description)

Now, visiting the /about redirect back to /github as we have passed github as id to / route.

Note: Flask uses Werkzeug to handle routing, and it give more weightage on normal routes than one with dynamic parts.

You can directly write the URL as we wrote in redirect() method or you can use url_for() method also. The first argument of this method should be the name of the function associated with the route and then the optional dynamic route variable. Following these rules, we can write the /github part of redirect(/github) as url_for("index", id="github").

Let's import url_for from flask and update the body of about() function. 

from flask import Flask, jsonify, redirect, url_for

app = Flask(__name__)

@app.route('/<id>')
def index(id):
  return f"The id is {id}"

@app.route('/about')
def about():
  return redirect(url_for("index", id="github"))

@app.errorhandler(404)
def file_not_found(error):
  return str(error.description)

Once we are done with application development, we should always deactivate the virtual environment using deactivate command in the terminal. 

$ deactivate

This command will deactivate the virtual environment for the application and also remove the venv entry from the terminal prompt.

Now, if you run the flask run command, you will get the following error. 

$ flask run
'flask' is not recognized as an internal or external command,
operable program or batch file.

In this case, you first need to activate the virtual environment using the command you already know (hint: mentioned in this article) now. Then running the above command wouldn't throw the error. But, you'll not get the development environment. That you need to set again (using the command you already know).
