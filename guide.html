Style Guide

Everybody have their own style guide for coding. Here is mine. I'm not forcing you to choose this style guide. But, if you are working with any of my projects you need to follow this style guide.

HTML

  1. Indent by two spaces at a time.
  2. Use only lowercase.
  3. Remove trailing whitespace.
  4. Use HTML5.
  5. Specify the encoding via HTML meta tag that help with CSS also - <meta charset="utf-8">.
  6. Valid the HTML whenever possible using online tool such as - W3C HTML Validator.
  7. Use semantic tags whenever possible.
  8. Do not close void elements.
  9. Omit the type/css and type/javascript attributes for style sheets and scripts.
  10. Use double quotation marks for quoting attributes values.
  11. Do not follow any rule strictly that affect the readability of code.

CSS

  1. Indent by two spaces at a time.
  2. Use only lowercase.
  3. Remove trailing whitespace.
  4. Valid the CSS whenever possible using online tool such as - W3C CSS Validator
  5. Use meaningful or generic name for id and classes.
  6. Avoid abbreviations when it create confusion. For example, atr for author make confusion with attribute. So, use author. attrib make sense for attribute.
  7. Avoid unnecessary ancestor selectors. For example, ul#active is not recommended.
  8. Use shorthand property whenever possible.
  9. Omit the unit specification after value 0.
  10. Omit the leading 0 in value.
  11. Use 3 characters hexadecimal notation whenever possible.
  12. Separate world in id and class names by hyphen.
  13. Use semicolon after every declaration.
  14. Use a space after a property name's colon.
  15. Use a space between last selector and opening curly brace for declaration block.
  16. Separate rules by new lines.
  17. Write declaration in alphabetical order.
  18. Do not follow any rule strictly that affect the readability of code.

JavaScript

  1. Indent by two spaces at a time.
  2. File name must be in lowercase and may include underscore(_) or hyphen(-).
  3. File name must ends with .js.
  4. Source files are encoded in utf-8.
  5. One statement per line.
  6. Semicolons are required.
  7. Remove trailing whitespace.
  8. Place the space around the operators except unary operators.
  9. Separate the keyword by one space from opening brace and closing brace.
  10. Use one space after comma(,) and colon(;).
  11. Horizontal alignment is discouragement.
  12. Use let or const for defining variables. Do not use var.
  13. Declare one variable per line.
  14. Span the object on multiple lines if needed using trailing comma.
  15. Identifiers ( except constants ) and functions name should be in lowercase and if needed then use underscore(_).
  16. Use arrow function notation for defining functions.
  17. Use double quotes for string values.
  18. Do not use Hungarian notation.
  19. Class name should be in CamelCase.
  20. Constant name should be in UPPER_CASE with underscore.
  21. If acronym is part of naming convention, treat as word. For example, XmlHttpRequest is correct than XMLHTTPRequest.

JavaScript (Node)

  1. Save the reference of required package into constant.
  2. Give same name to constant as package name except something unusual. For example, express for Express framework.
  3. Order of required package should be, system package/module, then third party package/module, and finally any custom package/module.
  4. Use new syntax whenever possible and available as part of update instead of depending on transpiler.
