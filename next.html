Next.js

Next.js is a React based framework that ease the building of fast web applications. Let's create a new folder with name company-website. 

$ mkdir company-website

Once the folder is created, go inside the created folder. 

$ cd company-website

We're going to init the npm and create the package.json file. 

$ npm init -y

I added -y flag to accept the default answers for the npm questions. As I mentioned earlier that the Next.js is a React based framework and due to this we need to install next as well as react and react-dom packages. 

$ npm i react react-dom next

You might need to wait for a couple of minutes to finish the installation process. Once the installation is done, open the package.json file and update the script tag with "dev": "next". 

{
  "name": "company-website",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "dev": "next"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "dependencies": {
    "next": "^10.1.3",
    "react": "^17.0.2",
    "react-dom": "^17.0.2"
  }
}

We're are now ready with setup and configuration.

Let's run the dev command as follows. 

$ npm run dev

> company-website@1.0.0 dev
> next

ready - started server on 0.0.0.0:3000, url: http://localhost:3000
Error: > Couldn't find a `pages` directory. Please create one under the project root

Ah! We got the error. But, the error is self-explanatory. It mentioned that pages directory or folder is not found, please create under the project root. Let's do it then. 

$ mkdir pages

Now, let's run the dev command again. 

$ npm run dev

> company-website@1.0.0 dev
> next

ready - started server on 0.0.0.0:3000, url: http://localhost:3000

Great! Our application is running at http://localhost:3000. Let's open this URL in browser. Oh! We get the 404 error. That's ok because we haven't define a home page or root route within application. Next.js support file-system base routing. In other words, you'll have routing like PHP or HTML where the page route is directly associated with file within pages folder. Let's create an index.js within pages folder and write the following code inside this file. 

const Home = () => {
  return <h1>Home</h1>;
};

export default Home;

Here we created a React functional component and exports it. You don't have to import React or ReactDOM. Just create a component and exports it. Don't worry about the component name. You can give any name you like (even ok to not give any name). But, giving a meaningful name is good idea.

Now, if we reload the page, we'll see the Home as header text in browser. Because, index.js points to index route which is root route or /.

Within pages folder, let's create another page with name about.js file. In this file, let's write the following code.

Now, if you go from / to /about or /about to /, you wouldn't not see the complete page refresh.

There are a couple of software engineering principles that we should or must follow and one of them is DRY or Don't Repeat Yourself. We're doing exactly opposite of it. We're duplicating the same <nav> tag in two different pages. Duplicating code has its own advantages in some special circumstances but not in our company-website application. What will happen if our application will have more pages and then we need to add one link within navigation? You get the point, right!

So, let's eliminate the repetitive code from our application by the use of layout. What we'll do is create Layout component and inside this Layout component, we'll have <nav>. Below, this <nav> we're going to render the content from the pages where we use Layout. For example, this is how we're going to use Layout component. 

<Layout>
  // Content goes here.
</Layout>

But, we can't create this Layout within pages folder. Why? Because, if we do it, then it'll be available at /layout. In this case, we can create this Layout as component within components folder outside the pages folder and under the company-website root folder. Just like we create a normal React component.

First, in company-website project root, let's create a components folder. 

$ mkdir components

Type {``} within this style tag. And then you can write the css you want e.g. literal string in JavaScript as script. See the font size is updated. In order to update the style of body, html and all other tags which are outside of this component but on this page, we need to add global attribute to styled JSX tag as follow.
