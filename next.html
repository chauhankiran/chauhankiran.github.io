Next.js

Next.js is a React based framework that ease the building of fast web applications. Let's create a new folder with name hello-world. 

$ mkdir hello-world

Once the folder is created, go inside the created folder. 

$ cd hello-world

Next, we're going to init the npm and create the package.json file. 

$ npm init -y

I added -y flag to accept the default answers for the npm questions. As I mentioned earlier that the Next.js is a React based framework and due to this we need to install next as well as react and react-dom packages as follows. 

$ npm i react react-dom next

You might need to wait for a couple of minutes to finish the installation process. Once the installation is done, open the package.json file and update the script tag with "dev": "next" as follows. 

{
  "name": "hello-world",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "dev": "next"
  },
  "keywords": [],
  "author": "",
  "license": "ISC",
  "dependencies": {
    "next": "^10.0.3",
    "react": "^17.0.1",
    "react-dom": "^17.0.1"
  }
}

We're are now ready with setup and configuration.

Let's run the dev command as follows. 

$ npm run dev

> hello-world@1.0.0 dev
> next

Error: > Couldn't find a `pages` directory. Please create one under the project root

Ah! We got the error. But, the error is self-explanatory. It mentioned that pages directory or folder is not found, please create under the project root. Let's do it then. 

$ mkdir pages

Now, let's run the dev command again. 

$ npm run dev

> hello-world@1.0.0 dev
> next

ready - started server on http://localhost:3000

Great! Our application is running at http://localhost:3000. Let's open this URL in browser. Oh! We get the 404 error. That's ok because we haven't define a home page or root route within application. Next.js support file-system base routing. In other words, you'll have routing like PHP or HTML where the page route is directly associated with file within pages folder. Let's create an index.js within pages folder and write the following code inside this file. 

const Home = () => {
  return (
    <h1>Home</h1>
  )
}

export default Home;

Here we created a React function component and exports it. You don't have to import React or ReactDOM. Just create a component and exports it. Don't worry about the component name. You can give any name you like. But, giving a meaningful name is good idea.

Now, if we reload the page, we'll see the Home as header text in browser. Because, index.js points to index route which is root route or /.

Within pages folder, let's create another page with name about.js file. In this file, let's write the following code. 

const About = () => {
  return (
    <h1>About</h1>
  )
}

export default About;

Now, if you visit /about page, you'll see About as header text in browser. Based on name of the file, you'll auto get the routing within pages folder. Got it? Let's add a navigation within both index.js and about.js files. Here is the updated content of index.js file.

        <a href="/about">About</a>
      </nav>
      <h1>Home</h1>
    </>

Now, if you click on About link from home page, it'll open About page and if you click on Home link from About page, it'll open Home page. But, did you notice the complete browser reload? We can eliminate this complete reload via Link component. Let's import this Link component from next/link in index.js file as follows. 

import Link from 'next/link'

const Home = () => {
  return (
    <>
      <nav>
        <a href="/about">About</a>
      </nav>
      <h1>Home</h1>
    </>
  )
}

export default Home;

Then wrap the <a> tag with <Link> tag as follows.

But, if you click on Home link from About page, you'll see complete page refresh. Magic of <Link> tag! Let's update the same change in about.js file as well.

There are a couple of software engineering principles that you should or must follow and one of them is DRY or Don't Repeat Yourself. We're doing exactly opposite of it. We're duplicating the same <nav> tag in two different pages. Duplicating code has its own advantages in some special circumstances but not in our hello-world application. What will happen if our application will have more pages and then we need to add one link within navigation? You get the point, right!

So, let's eliminate the repetitive code from our application by the use of layout. What we'll do is create Layout component and inside this Layout component, we'll have <nav>. Below, this <nav> we're going to render the content from the pages where we use Layout. For example, this is how we're going to use Layout component. 

<Layout>
  // Content goes here.
</Layout>

But, we can't create this Layout within pages folder. Why? Because, if we do it, then it'll be available at /layout. In this case, we can create this Layout as component within components folder outside the pages folder and under the project root folder. Just like we create a normal React component.

First, in project root, let's create a components folder. 

$ mkdir components

Then inside this created components folder, create a Layout.js file. Yeah, I intentionally gave the name Layout.js and not layout.js. Why? Because, it's a component and not the page. 

const Layout = () => {
  return (

  )
}

export default Layout;

Let's copy the common code between index.js and about.js file i.g. <nav> code. 

const Layout = () => {
  return (
    <nav>
      <Link href="/">
        <a>Home</a>
      </Link>
      &nbsp;
      <Link href="/about">
        <a>About</a>
      </Link>
    </nav>
  )
}

export default Layout;

We need to import the Link tag here in this Layout.js file.

const Layout = () => {

Now, we're going to wrap this <nav> tag within React fragment as follows. 

import Link from 'next/link'

const Layout = () => {
  return (
    <>
      <nav>
        <Link href="/">
          <a>Home</a>
        </Link>
        &nbsp;
        <Link href="/about">
          <a>About</a>
        </Link>
      </nav>
    </>
  )
}

export default Layout;

Why? Because, after <nav> tag, we're going to render the actual content of index.js or about.js file. In this case, we need to use props.children in Layout.js file as follows. 

import Link from 'next/link'

const Layout = (props) => {
  return (
    <>
      <nav>
        <Link href="/">
          <a>Home</a>
        </Link>
        &nbsp;
        <Link href="/about">
          <a>About</a>
        </Link>
      </nav>

      { props.children }
    </>
  )
}

export default Layout;

Let's use the concept of destructuring with props and accept children directly. 

import Link from 'next/link'

const Layout = ({ children }) => {
  return (
    <>
      <nav>
        <Link href="/">
          <a>Home</a>
        </Link>
        &nbsp;
        <Link href="/about">
          <a>About</a>
        </Link>
      </nav>

      { children }
    </>
  )
}

export default Layout;
